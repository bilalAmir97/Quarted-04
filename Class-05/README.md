**Prompt-Driven Development (PDD)**
Is approach mein hum AI ko instructions (prompts) dete hain aur wo code generate karta hai. Jitna acha prompt, utna acha code milega. Developer ka kaam prompt likhna hai, code likhna nahi.﻿​

Real-World Example: ChatGPT ya Gemini ko kehna "Ek login form banao React mein jo email aur password validate kare" - AI pura code bana dega.

**AI-Driven Development**
Puri development process mein AI lead karta hai. AI khud decisions leta hai, testing karta hai, bugs fix karta hai. Yahan AI sirf helper nahi, balke main driver hai jo development ko control karta hai.

Real-World Example: GitHub Copilot Workspace jahan AI khud code likhta hai, test cases banata hai, aur deployment bhi manage karta hai without human intervention.

**AI-Assisted Development**
Yahan developer lead karta hai aur AI uski help karta hai. AI suggestions deta hai, autocomplete karta hai, lekin final decision developer ka hota hai. Ye sabse common approach hai aaj kal.

Real-World Example: VS Code mein GitHub Copilot use karna - tum code likhte ho, wo next line suggest karta hai. Accept karo ya ignore karo, tumhari marzi.

**AI Native Development**
Jab application shuru se hi AI ko mind mein rakh kar banayi jaye. AI baad mein add nahi hota, balke foundation mein hota hai. Har feature AI-powered hoti hai from day one.

Real-World Example: Netflix ki recommendation system - pura platform AI ke saath design kiya gaya tha, baad mein add nahi kiya.

**AI Native IDE**
Ek development environment jo ground-up AI ke saath banaya gaya ho. Sirf AI plugin nahi, balke poora IDE AI capabilities ke saath design kiya gaya hai. Code editor khud AI se integrated hai.

Real-World Example: Cursor IDE - jo specifically AI development ke liye banaya gaya hai, jahan tum natural language mein instructions de sakte ho.

**AI Native Applications**
Wo applications jo shuru se AI ko core feature ki tarah use karti hain. Ye apps bina AI ke kaam nahi kar sakti kyunki AI unka main component hai, optional feature nahi.

Real-World Example: ChatGPT app, Midjourney, Grammarly - ye sab bina AI ke kaam hi nahi kar sakte kyunki AI hi inka main function hai.

**Spec-Driven Development (SDD):**
Is method mein pehle detailed specification (requirements) likhi jati hai ke kya banana hai. Phir AI ya developers us spec ke mutabiq code banate hain. Focus 'what' pe hai, 'how' pe nahi.

Real-World Example: Client kehta hai "Mujhe ek e-commerce site chahiye jo products show kare, cart ho, payment gateway ho" - ye specification hai. Phir AI isko samajh kar code banata hai.​

Behaviour-Driven Development (BDD)
Is approach mein hum application ki behaviour plain language mein likhte hain (Given-When-Then format mein). Phir tests us behaviour ke basis pe likhe jate hain. Non-technical log bhi samajh sakte hain.
